# 问题记录模板

## 基础模板

```markdown
### [P1] YYYY-MM-DD - 问题标题
- **问题**：具体问题描述
- **影响**：对开发的影响
- **建议**：初步解决思路
- **涉及文件**：相关的 Skill 或文档文件
- **风险评估**：低风险/中风险/高风险，说明原因
- **状态**：待处理
```

## 优先级说明

- **P0 (阻塞)**：严重影响开发进度，必须立即处理
- **P1 (重要)**：影响开发效率，项目结束后处理
- **P2 (优化)**：改善体验，定期批量处理

## 风险评估说明

- **低风险**：向后兼容，不影响现有功能
- **中风险**：可能影响部分功能，需要测试验证
- **高风险**：可能破坏现有功能，需要谨慎处理

## 记录示例

### 规范缺失类问题
```markdown
### [P1] 2026-01-17 - 代码生成缺少异常处理模板
- **问题**：使用 /proj-gen 生成的 Service 代码缺少统一的异常处理
- **影响**：每次都要手动添加 try-catch 和业务异常转换
- **建议**：在 Service 模板中增加标准异常处理模式
- **涉及文件**：.claude/skills/proj-gen/templates/service.md
- **风险评估**：低风险，只是增加模板内容，向后兼容
- **状态**：待处理
```

### 流程问题类
```markdown
### [P1] 2026-01-17 - 需求确认环节不够细致
- **问题**：需求分析时经常遗漏边界条件
- **影响**：开发过程中频繁返工
- **建议**：在需求分析模板中增加边界条件检查清单
- **涉及文件**：.claude/skills/proj-analyze-req/SKILL.md
- **风险评估**：低风险，只是增加检查项，不改变现有流程
- **状态**：待处理
```

### 工具使用类
```markdown
### [P2] 2026-01-17 - Skill 索引查找不便
- **问题**：需要在 CLAUDE.md 中查找对应的 Skill 说明
- **影响**：降低使用效率
- **建议**：考虑增加 /proj-help 命令快速查看 Skill 列表
- **涉及文件**：新增功能
- **风险评估**：中风险，需要新增 Skill，可能影响现有命令体系
- **状态**：待处理
```

### 高风险问题示例
```markdown
### [P0] 2026-01-17 - 核心流程执行异常
- **问题**：/proj-gen 在某些情况下生成失败，导致开发阻塞
- **影响**：无法正常生成代码，严重影响开发进度
- **建议**：增加错误处理和重试机制，完善异常提示
- **涉及文件**：.claude/skills/proj-gen/SKILL.md
- **风险评估**：高风险，涉及核心流程修改，需要充分测试
- **状态**：待处理
```
