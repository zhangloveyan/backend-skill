# 任务管理

## 任务文档位置

```
docs/TASK.md
```

---

## 任务文档格式

```markdown
# 开发任务

## 当前任务

### {日期} {功能名称}

**方案文档：** `docs/方案/{功能名称}.md`（如有）

**任务清单：**

- [ ] 任务1
- [~] 任务2（进行中）
- [x] 任务3（已完成）

**提交记录：**

- `{commit-hash}` {type}: {message}

---

## 历史任务

（已完成的任务归档到这里）
```

---

## 任务状态标记

| 标记 | 状态 | 说明 |
|------|------|------|
| `[ ]` | 待处理 | 未开始 |
| `[~]` | 进行中 | 正在开发 |
| `[x]` | 已完成 | 开发完成 |
| `[!]` | 阻塞 | 有问题待解决 |
| `[-]` | 取消 | 不再需要 |

---

## 操作指令

### 创建任务

**触发：** 方案确认后

**标准任务顺序：**
```markdown
- [ ] 创建数据库表
- [ ] 创建 Entity 实体类
- [ ] 创建 Mapper 接口和 XML
- [ ] 创建 Service 接口
- [ ] 创建 Service 实现类
- [ ] 创建 Request DTO
- [ ] 创建 Response DTO
- [ ] 创建 Controller（Web端）
- [ ] 创建 Controller（App端）
- [ ] 自测验证
```

### 更新任务状态

- 开始任务前：`[ ]` → `[~]`
- 完成任务后：`[~]` → `[x]`
- 遇到阻塞：`[~]` → `[!]` 并说明原因
- 任务取消：`[ ]` → `[-]`

**注意：** 同一时间只有一个任务为 `[~]`，完成后立即更新。

### 查看任务

**汇报格式：**
```
当前任务：{功能名称}
进度：{已完成}/{总数}
状态：
- [x] 已完成任务1
- [~] 进行中任务2
- [ ] 待处理任务3
```

### 归档任务

功能开发完成后，将任务块从「当前任务」移动到「历史任务」。

---

## 注意事项

1. **及时更新** - 状态变化后立即更新文档
2. **保持简洁** - 任务描述简明扼要
3. **记录提交** - 便于追溯和回滚
4. **定期归档** - 完成的任务移到历史区
